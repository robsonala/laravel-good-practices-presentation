Middlewares

 

When should I use it?
Always you need to apply the same logic in a Request and/or Response for more than one route.

Applied on Request


The code is executed before the action.

Example

class CheckAge
{
    /**
     * Handle an incoming request.
     *
     * @param  \Illuminate\Http\Request  $request
     * @param  \Closure  $next
     * @return mixed
     */
    public function handle($request, Closure $next)
    {
        if ($request->age <= 200) {
            return redirect('home');
        }

        return $next($request);
    }
}

Applied on Response


The code is executed after the action.

Example

class AfterMiddleware
{
    public function handle($request, Closure $next)
    {
        $response = $next($request);

        // Perform action

        return $response;
    }
}

Eloquent: Collection

 

All database query made using Eloquent(Model) is an instance of Collection
Collections on Laravel has loads of useful helpers that should be used always necessary.

General tips

 

• Google/Stackoverflow is our friend, but always check the Official documentation.
• If you need a specific feature check if already exists (https://packagist.org/) before create it.
• Use ENUM or CONST in a class when you are using a fixed value.
Example:

class Test {
    const MAX_ATTEMPT = 5;
}
...
file1.php
if ($qty > Test::MAX_ATTEMPT)
...
file2.php
while ($count < Test::MAX_ATTEMPT)
